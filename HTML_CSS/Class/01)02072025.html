In programming, a rubber duck refers to the rubber duck debugging technique,
where a programmer explains their code line by line to an inanimate object,
like a rubber duck, to help identify and fix errors. This method forces the
programmer to articulate the code's logic, which can expose flaws in their
thinking or reveal overlooked issues
